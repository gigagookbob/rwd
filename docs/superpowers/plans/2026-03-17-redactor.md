# Redactor Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a `redactor` module that detects sensitive information (API keys, tokens, private IPs, etc.) and masks them with `[REDACTED:TYPE]` before LLM API transmission.

**Architecture:** An independent module `src/redactor/` performs regex-based masking on prompt text. Called inside `analyzer/mod.rs` after `build_prompt()` and before `call_api()`. Toggled on/off via `[redactor] enabled` in config.toml.

**Tech Stack:** Rust 2024 Edition, `regex` crate, `LazyLock` (std::sync), `BTreeMap` (std::collections)

**Spec:** `docs/superpowers/specs/2026-03-17-redactor-design.md`

```rust
// The redactor module detects and masks sensitive information before LLM API transmission.
// Applied to build_prompt() results in the analyzer module to block external leakage.

pub mod patterns;

use std::collections::BTreeMap;

/// Masking result summary.
/// Uses BTreeMap to guarantee alphabetical sorting of type names (HashMap has no ordering guarantee).
pub struct RedactResult {
    pub total_count: usize,
    pub by_type: BTreeMap<String, usize>,
}

impl RedactResult {
    /// Empty result for when redactor is disabled.
    pub fn empty() -> Self {
        Self {
            total_count: 0,
            by_type: BTreeMap::new(),
        }
    }

    /// Merges multiple RedactResults (combines Claude + Codex results).
    /// The entry() API inserts a default if the key is absent, or accesses the existing value (Rust Book Ch.8).
    pub fn merge(&mut self, other: RedactResult) {
        self.total_count += other.total_count;
        for (key, count) in other.by_type {
            *self.by_type.entry(key).or_insert(0) += count;
        }
    }

    /// Generates a summary string like "API_KEY: 3, BEARER_TOKEN: 1".
    pub fn format_summary(&self) -> String {
        self.by_type
            .iter()
            .map(|(k, v)| format!("{k}: {v}"))
            .collect::<Vec<_>>()
            .join(", ")
    }
}

/// Detects sensitive information in text and replaces it with [REDACTED:TYPE].
/// Patterns are initialized via LazyLock, so this function never fails.
///
/// Note: Rules are applied in order. Patterns must be designed so that
/// a preceding rule's replacement text is not matched by a following rule.
pub fn redact_text(text: &str) -> (String, RedactResult) {
    let rules = patterns::builtin_rules();
    let mut result_text = text.to_string();
    let mut by_type: BTreeMap<String, usize> = BTreeMap::new();
    let mut total_count: usize = 0;

    for rule in rules {
        // Count matches with find_iter() first, then replace only if there are matches.
        // Passing &str to replace_all() avoids the capture-based code path, which is more efficient.
        let count = rule.pattern.find_iter(&result_text).count();
        if count > 0 {
            let replacement = format!("[REDACTED:{}]", rule.name);
            result_text = rule
                .pattern
                .replace_all(&result_text, replacement.as_str())
                .into_owned();
            *by_type.entry(rule.name.to_string()).or_insert(0) += count;
            total_count += count;
        }
    }

    (result_text, RedactResult { total_count, by_type })
}
```

```rust
// Defines built-in sensitive information detection patterns.
// LazyLock is a lazily initialized static variable that initializes on first access (std::sync::LazyLock).
// Regex compilation is expensive, so it's done once and reused.

use regex::Regex;
use std::sync::LazyLock;

/// Pattern kind — FixedPrefix can be replaced with Aho-Corasick in the future.
/// Currently both kinds use Regex; kind serves as metadata only.
#[allow(dead_code)]
pub enum PatternKind {
    FixedPrefix,
    Regex,
}

/// Represents a single masking rule.
pub struct RedactorRule {
    pub name: &'static str,
    #[allow(dead_code)]
    pub kind: PatternKind,
    pub pattern: &'static Regex,
}

/// Returns the list of built-in patterns.
/// Each pattern is compiled once via LazyLock.
pub fn builtin_rules() -> Vec<RedactorRule> {
    // LazyLock<Regex>: calls Regex::new() on first access to compile.
    // expect() panics on regex syntax errors (programming error) — acceptable since these are not runtime inputs.
    static API_KEY: LazyLock<Regex> =
        LazyLock::new(|| Regex::new(r"\bsk-[a-zA-Z0-9]{20,}\b").expect("API_KEY regex"));
    static AWS_KEY: LazyLock<Regex> =
        LazyLock::new(|| Regex::new(r"\bAKIA[0-9A-Z]{16}\b").expect("AWS_KEY regex"));
    static GITHUB_TOKEN: LazyLock<Regex> =
        LazyLock::new(|| Regex::new(r"\bgh[ps]_[a-zA-Z0-9]{36,}\b").expect("GITHUB_TOKEN regex"));
    static SLACK_TOKEN: LazyLock<Regex> =
        LazyLock::new(|| Regex::new(r"\bxox[bpsa]-[a-zA-Z0-9\-]+\b").expect("SLACK_TOKEN regex"));
    static BEARER_TOKEN: LazyLock<Regex> =
        LazyLock::new(|| Regex::new(r"Bearer\s+[a-zA-Z0-9\-._~+/]+=*").expect("BEARER_TOKEN regex"));
    static ENV_SECRET: LazyLock<Regex> = LazyLock::new(|| {
        Regex::new(r#"(?i)(password|secret|api_key)\s*=\s*["'][^"']+["']"#).expect("ENV_SECRET regex")
    });
    static PRIVATE_IP: LazyLock<Regex> = LazyLock::new(|| {
        Regex::new(r"\b(10\.\d+\.\d+\.\d+|172\.(1[6-9]|2\d|3[01])\.\d+\.\d+|192\.168\.\d+\.\d+)\b")
            .expect("PRIVATE_IP regex")
    });
    static PRIVATE_KEY: LazyLock<Regex> = LazyLock::new(|| {
        Regex::new(r"-----BEGIN[A-Z ]*PRIVATE KEY-----").expect("PRIVATE_KEY regex")
    });

    vec![
        RedactorRule { name: "API_KEY", kind: PatternKind::FixedPrefix, pattern: &API_KEY },
        RedactorRule { name: "AWS_KEY", kind: PatternKind::FixedPrefix, pattern: &AWS_KEY },
        RedactorRule { name: "GITHUB_TOKEN", kind: PatternKind::FixedPrefix, pattern: &GITHUB_TOKEN },
        RedactorRule { name: "SLACK_TOKEN", kind: PatternKind::FixedPrefix, pattern: &SLACK_TOKEN },
        RedactorRule { name: "BEARER_TOKEN", kind: PatternKind::Regex, pattern: &BEARER_TOKEN },
        RedactorRule { name: "ENV_SECRET", kind: PatternKind::Regex, pattern: &ENV_SECRET },
        RedactorRule { name: "PRIVATE_IP", kind: PatternKind::Regex, pattern: &PRIVATE_IP },
        RedactorRule { name: "PRIVATE_KEY", kind: PatternKind::Regex, pattern: &PRIVATE_KEY },
    ]
}
```
